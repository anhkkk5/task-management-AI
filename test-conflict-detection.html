🧪 Test Conflict Detection & Scheduled Status

Hướng dẫn test tính năng phát hiện xung đột lịch và trạng thái "Đã lên lịch"

📋 Test Case 1: Tạo Task và AI Schedule

1Tạo Task A

Vào trang Tasks: http://localhost:5173/tasks

Click "Thêm công việc" và nhập:

Tiêu đề: Test Task A - học tiếng anh Thời gian dự kiến: 2h Mục tiêu/ngày: 1h-1h Hạn chót: 11/3/2026

✅ Expected:

  • Task được tạo với status = Chưa xử lý
  • Hiện trong danh sách tasks

2AI Schedule Task A

1. Click "AI Tối Ưu Lịch"

2. Chọn Task A

3. Click "Phân tích và tạo lịch"

4. Đợi AI phân tích (5-10 giây)

5. Click "Áp dụng lịch trình"

✅ Expected:

  • Success message: "Đã lưu lịch trình"
  • Task A status tự động chuyển sang Đã lên lịch ⭐
  • Task A có thời gian: VD "08:00 - 09:00"

🔍 Check Backend Console:

Bạn sẽ thấy log:

[Conflict Detection] Found 0 scheduled tasks [Scheduler] Task A scheduled at 08:00-09:00

🔍 Test Case 2: Conflict Detection

1Tạo Task B

Tiêu đề: Test Task B - học code Thời gian dự kiến: 2h Mục tiêu/ngày: 1h-1h Hạn chót: 11/3/2026

✅ Expected:

  • Task B được tạo với status = Chưa xử lý

2AI Schedule Task B (Conflict Detection Test)

1. Click "AI Tối Ưu Lịch"

2. Chọn ONLY Task B (không chọn Task A)

3. Click "Phân tích và tạo lịch"

⭐ QUAN TRỌNG - Check Backend Console:

[Conflict Detection] Found 1 scheduled tasks [Scheduler] Date 2026-03-07: 1 busy slots [Scheduler] Task B scheduled at 09:15-10:15 (avoiding Task A)

4. Click "Áp dụng lịch trình"

✅ Expected:

  • Task B được schedule VÀO THỜI GIAN KHÁC với Task A
  • Task B status = Đã lên lịch
  • KHÔNG có overlap với Task A

📅 Expected Timeline:

08:00 - 09:00 Task A: học tiếng anh ✓
09:15 - 10:15 Task B: học code ✓ (có 15 phút buffer)

🎉 Verify NO CONFLICT:

  • Task A end time: 09:00
  • Task B start time: 09:15
  • Gap: 15 minutes (buffer) ✓
  • NO OVERLAP! 🎉

📊 Test Case 3: Multiple Scheduled Tasks

1Tạo Task C

Tiêu đề: Test Task C - tiếng hàn Thời gian dự kiến: 1h Mục tiêu/ngày: 1h-1h Hạn chót: 11/3/2026

2AI Schedule Task C

1. Click "AI Tối Ưu Lịch"

2. Chọn ONLY Task C

3. Click "Phân tích và tạo lịch"

⭐ Check Backend Console:

[Conflict Detection] Found 2 scheduled tasks [Scheduler] Date 2026-03-07: 2 busy slots [Scheduler] Task C scheduled at 10:30-11:30 (avoiding Task A & B)

📅 Expected Timeline:

08:00 - 09:00 Task A: học tiếng anh ✓
09:15 - 10:15 Task B: học code ✓
10:30 - 11:30 Task C: tiếng hàn ✓ (tránh cả A và B)

🎨 Test Case 4: Status Dropdown

1Click vào Status Tag

Trong danh sách tasks, click vào status tag của Task A

✅ Expected Options:

  • Chưa xử lý (gray)
  • Đã lên lịch (blue) ⭐ NEW
  • Đang làm (orange)
  • Hoàn thành (green)
  • Đã hủy (red)

2Change Status

1. Click "Đang làm"

2. Đợi loading

✅ Expected:

  • Success message: "Đã cập nhật trạng thái"
  • Status tag chuyển sang Đang làm
  • Task list tự động refresh

3Change Back to Scheduled

1. Click vào status tag lại

2. Click "Đã lên lịch"

✅ Expected:

  • Status chuyển về Đã lên lịch
